# Файловий менеджер

File manager — одна з найглибших зон у поточному коді. Хендлер у `src/handlers/files.rs` великий, бо покриває навігацію, редагування, uploads, архіви й безпекові перевірки.

## Основні можливості

- перегляд папок і файлів із breadcrumb navigation
- full-screen редагування текстових і конфігураційних файлів
- створення файлів і директорій
- rename, copy, move, paste і delete
- drag-and-drop upload із прогресом по файлах
- створення архівів `.tar.gz`
- розпакування `.tar.gz`, `.tar.bz2`, `.tar.xz`, `.zip`, `.jar`, `.rar`, `.7z`, `.gz`, `.bz2` і `.xz`

## Обізнаність щодо типів файлів

Upstream handler мапить сильно більше сотні типів файлів і exact filenames у категоризовані іконки. Це охоплює:

- код і скрипти
- конфігураційні формати на кшталт YAML, TOML, `.env` та INI
- архіви та JAR-файли
- зображення, аудіо й відео
- сертифікати та ключі
- текст, markdown, CSV, логи та офісні документи

## Захист і safety controls

- path traversal перевірки прив'язують усі операції до кореня volume
- uploads спочатку стрімляться в тимчасовий файл, а вже потім переносяться у volume
- archive extraction за потреби виконується через контрольований helper-сценарій
- bulk operations усе одно валідовують кожен path проти межі volume

## Практичний висновок

Це не просто "браузер файлів", а радше робоча панель для volume-даних контейнера, тобто одна з ключових частин щоденного використання Yunexal Panel.
